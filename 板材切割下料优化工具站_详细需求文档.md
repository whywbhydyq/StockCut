## 0. 文档结论

本项目应优先做成一个面向木工、DIY、柜体制作、小型工坊、广告亚克力切割、金属管材/铝型材下料用户的轻量工具站。核心价值不是“解释什么是下料优化”，而是让用户输入真实材料尺寸和零件尺寸后，立即得到可打印、可执行、包含锯缝补偿的切割布局。

最小可行产品不是单一计算器，而是一个轻量工具矩阵：

1. **Sheet Cutting Optimizer**：2D 板材矩形排版，适用于 plywood、MDF、melamine、acrylic、glass、metal sheet。
2. **Linear Cutting Optimizer**：1D 线材/管材/型材下料，适用于 lumber、steel tube、aluminum extrusion、PVC pipe、rebar。
3. **Saw Kerf Calculator**：锯缝补偿与尺寸可行性解释。
4. **Plywood Yield / Waste Calculator**：出材率、废料率、材料张数估算。
5. **可打印 / PDF 输出**：把结果从“网页答案”转化为“工坊可执行文件”。

第一版要保守，不追求工业最优、CNC、异形 nesting、账号系统、云端项目管理。项目成败的关键是：**锯缝是否可信、单位是否稳定、排版图是否能看懂、输出是否能拿去切、SEO 页面是否承接真实长尾搜索意图。**

## 1. 背景与依据

### 1.1 项目来源

上传底稿将“板材切割 / 一维二维下料优化排版”列为新开独立站第一优先方向，理由包括：

- 工具意图极强。
- 交互输出强。
- AI 难以替代实时交互式 2D 排版图、锯缝补偿、PDF 下料单和切割顺序可视化。
- 材料、锯片、电锯、夹具、卷尺、五金、木工图纸、课程等商业邻接自然。
- 最脆弱点是算法可信度、锯缝与单位边界。

底稿给出的核心输入包括：原料尺寸、目标零件宽/高/数量/名称、是否允许旋转、锯缝 Kerf、单位、1D/2D 模式。核心输出包括：2D 锯路切割线图、Guillotine Cut 布局、使用板材张数、出材率、剩余废料清单、步骤式切割顺序、可打印下料单。

### 1.2 外部需求与竞品信号

已观察到的外部信号包括：

- OptiCutter 将 sheet/roll 长宽、cutting kerf、required panel sizes、CSV/Excel 导入、多种单位、grain direction、edge banding、trim edges、price、waste/cost minimization 等作为核心功能。
- CutList Evolution / SmartCut 的公开文档和社区讨论反复出现 number format、part rotation、stock matching、blade width、linear & sheet materials、kerf、banding、trim、grain orientation、PDF、estimate costs、mobile & desktop 等需求。
- 木工社区和专业内容中，用户真实动机不是“好奇算法”，而是“我有 plywood 项目，要尽量少浪费昂贵材料，并把切割图拿到工坊执行”。
- 专业木工内容强调 cut list 是设计与制造之间的桥梁，手工切割清单和手工排版容易出错、耗时、浪费材料。

因此，本项目的真实需求基础成立。

## 2. 产品定位

### 2.1 一句话定位

**A free online cut list optimizer that turns your plywood, MDF, acrylic, lumber, pipe, or metal stock into printable cutting layouts with kerf included.**

中文理解：

**一个免费的在线下料优化工具，把板材、木条、钢管、铝型材等原料和零件清单转成包含锯缝补偿的可打印切割布局。**

### 2.2 用户不是谁

本项目第一版不服务以下用户：

- 需要 CNC 后处理、DXF、G-code 的工厂。
- 需要 ERP、库存、订单排产、条码、设备联动的企业用户。
- 需要异形 nesting、曲线切割、复杂多边形排样的工业用户。
- 需要 AI 自动设计整套柜体、家具、房屋结构的用户。
- 需要工程担保或法律责任背书的生产系统。

### 2.3 用户是谁

第一版服务以下人群：

| 用户类型 | 典型场景 | 核心诉求 |
|---|---|---|
| DIY 木工 | 书架、柜子、抽屉、工作台、收纳架 | 少买板、少浪费、切割图能打印 |
| 家庭装修用户 | closet shelf、pantry shelf、garage storage | 用 4x8 plywood/MDF 算出所需张数和切法 |
| 小型木工作坊 | 柜体、家具、小批量订单 | 减少排版时间和材料浪费 |
| 广告/展示制作 | 亚克力板、PVC 板、展示架 | 矩形板件快速排版 |
| 金属加工/DIY | steel tube、aluminum extrusion、rebar | 1D 型材切割最少废料 |
| 学校/创客空间 | 多人项目、材料有限 | 快速给出可执行下料单 |
| Etsy/小卖家 | 小批量木制品、激光切割前准备 | 降低重复项目材料成本 |

## 4. Jobs To Be Done

### JTBD-1：DIY 木工的 4x8 胶合板排版

当我准备做一个书架、柜子或抽屉项目时，  
我想把所有 plywood 零件尺寸输入进去，  
这样我可以知道要买几张 4x8 板，并拿到一张能打印的切割图，减少浪费和切错。

成功标准：

- 3 分钟内完成输入。
- 得到每张板的布局图。
- 清楚显示每个零件标签。
- 可打印。
- kerf 已计入。

### JTBD-2：小工坊的快速下料单

当我接到一个小型柜体/家具订单时，  
我想把零件表粘贴到网页里，  
这样我可以快速生成下料图、材料数量和废料率，而不是手工用 CAD/Excel 排版。

成功标准：

- 支持从 Excel 粘贴。
- 支持按材料分组。
- 输出 sheet count、yield、waste。
- 可导出 PDF/CSV。
- 错误行清楚提示。

### JTBD-3：线材/管材切割

当我有一批 steel tube、aluminum extrusion、PVC pipe 或 lumber 长度时，  
我想知道如何从标准长度原料中切出所有段，  
这样我可以少买料、少浪费，并把每根 stock 的切割顺序打印出来。

成功标准：

- 输入 stock length、part length、quantity、kerf。
- 输出每根 stock 的段落排列。
- 显示 offcut。
- 显示 waste rate。
- 支持同尺寸多根原料。

### JTBD-4：锯缝与可行性确认

当我不确定尺寸刚好能不能切出来时，  
我想快速验证 kerf 是否导致尺寸不可行，  
这样我不会买错板或切错第一刀。

成功标准：

- 可以独立使用 Saw Kerf Calculator。
- 明确解释 blade thickness / kerf。
- 对临界尺寸给出警告。
- FAQ 解释常见陷阱。

## 5. MVP 范围

### 5.1 P0 必须做

P0 是第一版上线前必须完成的功能，缺失会显著降低真实使用价值。

#### P0-1：Sheet Cutting Optimizer

必须支持：

- 2D 矩形板材排版。
- 原料板材尺寸输入。
- 零件名称、宽、高、数量输入。
- 是否允许旋转。
- kerf 设置。
- 单位选择。
- 自动计算所需板材张数。
- 输出每张板的布局图。
- 输出 sheet count、yield rate、waste rate、offcuts。
- 输出基本 cut sequence。
- 打印。
- PDF 导出。
- 行级错误提示。

#### P0-2：Linear Cutting Optimizer

必须支持：

- 1D 标准长度原料下料。
- 原料长度、数量或无限数量。
- 目标段长度、数量、名称。
- kerf 设置。
- 输出每根 stock 的切割段。
- 输出 offcut。
- 输出 waste rate。
- 打印/PDF。

#### P0-3：Saw Kerf Calculator

必须支持：

- 输入 blade kerf。
- 输入 cut count。
- 计算总损耗。
- 解释 finished size 与 raw stock size 的区别。
- 给出常见示例，例如 “two 24 in panels from 48 in sheet”。

#### P0-4：单位系统

必须支持：

- mm
- cm
- inch decimal
- inch fraction，例如 15 3/4
- feet + inches，例如 4 ft 8 in 或 4' 8"

内部统一转换为浮点数或整数微单位，显示时再格式化。

#### P0-5：导出和打印

必须支持：

- 浏览器打印样式。
- PDF 导出。
- 结果复制摘要。
- PNG/SVG 导出可放 P1，但 PDF/Print 必须 P0。

#### P0-6：基础 SEO 页面

必须上线：

- 首页 / 主工具页。
- Sheet Cutting Optimizer 页面。
- Linear Cutting Optimizer 页面。
- Saw Kerf Calculator 页面。
- 10 个长尾落地页。
- About。
- Privacy。
- Terms。
- Disclaimer。
- Sitemap。
- robots.txt。

### 5.2 P1 第二阶段

P1 是上线后根据 GSC、GA、用户行为和错误反馈决定的增强项。

- CSV 导入。
- Excel 文件导入。
- CSV 导出。
- JSON 项目保存/恢复。
- 本地浏览器保存最近项目。
- Grain direction。
- Stock edge trim。
- Edge banding 标记。
- Multiple stock sizes。
- Material price / cost estimate。
- Offcut reuse。
- Strategy toggle：least sheets / least waste / fewer cuts。
- 结果图缩放、平移。
- 移动端 shop mode。
- 分享链接，最好不含敏感数据，或仅编码在 URL hash。

### 5.3 P2 后续可做

- Manual drag adjustment。
- Grain matching groups。
- Angle cuts for metal/pipe。
- Identical cutting layouts。
- DXF export。
- CNC/router mode。
- Inventory/offcut library。
- Project quote。
- 批量项目模板。
- 多语言页面。
- affiliate comparison pages。

### 5.4 第一版明确不做

- 异形 nesting。
- 圆形、三角形、多边形零件。
- 曲线切割。
- CNC G-code。
- 工业 DXF。
- 账号系统。
- 云端保存。
- 订单排产。
- ERP。
- 设备联动。
- 自动报价系统。
- AI 设计柜体。
- 复杂材料库存。
- 用户上传敏感商业文件到服务器保存。

- 每根 stock 的 offcut。
- 总废料长度。
- waste rate。
- kerf included。
- printable cutting list。

### 8.3.3 算法要求

第一版可以使用启发式算法：

- Sort segments descending by length。
- Best Fit Decreasing：把当前 segment 放入剩余长度最接近但足够的 stock。
- 每个切割段之间扣除 kerf。
- 对最后一个段是否扣 kerf 要明确：实际从 stock 上分离每段需要切割；对于最后剩余 offcut，通常每切一段消耗一次 kerf。实现中应采用保守模型，并在说明中解释。

伪代码：

```ts
segments = expandByQuantity(parts).sort(descLength)
stocks = []

for segment in segments:
  bestStock = findStockWithMinimumRemainingAfterPlacement(segment.length + kerf)
  if bestStock exists:
    place segment
  else:
    create new stock
    place segment

calculate offcuts
```

## 8.4 Saw Kerf Calculator

### 8.4.1 用户问题

- What is saw kerf?
- How much material does a blade remove?
- Do I need to include kerf in a cut list?
- Why can't I cut two 24 inch panels from a 48 inch sheet?
- How many cuts can I make before kerf matters?

### 8.4.2 输入

| 字段 | 类型 | 示例 |
|---|---|---|
| Kerf / blade thickness | dimension | 1/8 in, 3 mm |
| Number of cuts | integer | 10 |
| Target finished part size | dimension | 24 in |
| Stock size | dimension | 48 in |

### 8.4.3 输出

- Total kerf loss = kerf × number of cuts。
- Required raw length/width = sum(finished parts) + kerf × cuts。
- 可行 / 不可行判断。
- 示例解释。

### 8.4.4 公式

```text
Total Kerf Loss = Kerf × Number of Cuts
Required Stock Length = Sum(Finished Part Lengths) + Total Kerf Loss
Yield Rate = Total Finished Part Area / Used Stock Area × 100%
Waste Rate = 100% - Yield Rate
```

### 8.4.5 FAQ 示例

**Can I cut two 24 inch panels from a 48 inch sheet?**  
Usually no, if both panels must be exactly 24 inches and a cut is required between them. The blade removes material equal to the kerf, so the required width is 24 + kerf + 24.

## 9. 算法与计算逻辑

### 9.1 2D 排版问题定义

输入：

- 一个或多个 stock rectangles。
- 多个 required part rectangles。
- kerf。
- rotation constraints。
- optional trim margins。
- optional material groups。

输出：

- 每个 part 放在哪张 stock sheet。
- 每个 part 的 x/y/width/height/rotation。
- 未放置 parts。
- offcuts。
- metrics。

### 9.2 2D MVP 算法策略

MVP 推荐实现两种策略，先用可解释、稳定、便于输出切割步骤的方案：

#### Strategy A：Guillotine recursive split

优点：

- 切割路径更接近实际手工下料。
- 容易输出切割顺序。
- 更适合 table saw / track saw / circular saw。
- 容易解释。

缺点：

- 不一定最省料。

#### Strategy B：Shelf / Skyline packing

优点：

- 实现相对简单。
- 对许多矩形场景表现可接受。
- 便于快速出图。

缺点：

- 切割顺序不一定自然。
- 对极端尺寸可能浪费。

第一版建议默认使用 practical guillotine layout，文案不要承诺 global optimum。可以显示：

> This tool generates a practical optimized layout. For complex industrial nesting, verify the plan before cutting.

### 9.3 排序策略

尝试多轮排序并选择最佳结果：

- 面积从大到小。
- 长边从大到小。
- 宽边从大到小。
- 不允许旋转件优先。
- 高数量重复件成组。
- 材料分组独立优化。

选择指标：

1. 最少使用 stock 数量。
2. 更高 yield rate。
3. 更少 unplaced parts。
4. 更少切割步骤。
5. 更大的可用 offcuts。

### 9.4 Kerf 处理

必须统一规则，避免不同模块不一致。

在 2D 布局中：

- part 的 finished dimension 不变。
- part 与 part 之间必须留出 kerf。
- part 与 trim 后可用区域边界之间不额外扣 kerf，除非需要从原始边修边。
- 如果设置 edge trim，则先扣除 trim，再在剩余区域排版。
- 视觉图显示 part finished area 和 kerf/cut line。

### 9.5 Offcut 处理

P0：

- 输出剩余可识别矩形区域。
- 标记尺寸。
- 不保证 offcut 最优复用。

P1：

- 允许用户把 offcuts 作为 stock 输入。
- 支持 “prefer offcuts first”。

### 9.6 线材 1D 算法

P0：

- Best Fit Decreasing。
- 支持有限/无限 stock。
- kerf 每次切割扣除。
- 输出每根 stock 的 segment order。
- 如果 segment > stock length，则标记 unplaced。

P1：

- 支持相同切割布局合并，例如 “Cut this pattern 8 times”。
- 支持目标：minimize waste / minimize stock count。
- 支持 offcut minimum usable length。

### 9.7 算法边界声明

所有页面和 PDF 应包含短免责声明：

- Results are generated from the dimensions you enter.
- Always verify the layout, kerf, grain direction, and machine setup before cutting.
- This tool is for planning and estimation, not a substitute for professional fabrication review.

## 18. 测试用例

### 18.1 单位解析测试

| 输入 | 单位模式 | 期望 |
|---|---|---|
| `2440` | mm | 2440mm |
| `96` | inch decimal | 96in |
| `15.75` | inch decimal | 15.75in |
| `15 3/4` | inch fraction | 15.75in |
| `1' 3 3/4"` | feet inches | 15.75in |
| `3,000` | mm | error 或提示 |
| `10,5` | decimal | error 或提示 |

### 18.2 Kerf 测试

| 场景 | 输入 | 期望 |
|---|---|---|
| 48 in 板切两个 24 in | stock 48, parts 24+24, kerf 1/8 | 不可行 |
| 48.125 in 板切两个 24 in | stock 48.125, kerf 1/8 | 可行 |
| 0 kerf | kerf 0 | 可行性按无损计算 |
| kerf 大于零件宽 | part 2mm, kerf 3mm | 警告 |

### 18.3 2D 排版测试

| 场景 | 期望 |
|---|---|
| 单个零件小于板材 | 成功放置 |
| 单个零件大于板材 | unplaced |
| 允许旋转后可放入 | rotated = true |
| 禁止旋转无法放入 | unplaced |
| 多数量重复件 | 展开后全部尝试放置 |
| stock 数量有限不足 | 输出不足警告 |
| edge trim 后不可放置 | 提示 usable area 不足 |

### 18.4 1D 排版测试

| 场景 | 期望 |
|---|---|
| 120 in stock，三个 40 in，kerf 0 | 一根刚好 |
| 120 in stock，三个 40 in，kerf 1/8 | 一根不够 |
| segment 超过 stock | unplaced |
| quantity 100 | 计算稳定 |
| offcut 很小 | 正确显示 |

### 18.5 打印/PDF 测试

- PDF 包含所有 sheet。
- 打印样式隐藏广告。
- 黑白打印仍能区分 parts。
- 长 label 不溢出。
- 多页 PDF 顺序正确。
- 移动端生成 PDF 不崩溃。

## 19. 指标与验收

### 19.1 功能验收

### 19.2 90 天观察指标

| 指标 | 目标/止损线 |
|---|---|
| 索引页面数 | 20–30 页完成索引 |
| GSC 日曝光 | 90 天后仍低于 200/day 需复盘 |

| 错误反馈 | 每周算法错误处理超过 5 小时需收缩范围 |
| Affiliate 点击 | 完全无承接需调整放置或方向 |
| AdSense RPM | 仅作观察，不作为早期唯一判断 |

### 19.3 继续投入条件

满足任意两项可继续扩展：

- GSC 持续增长。
- Optimize 点击稳定。
- PDF/Print 使用明显。
- 用户通过搜索进入多个长尾页。
- affiliate 有点击。

### 19.4 止损条件

符合以下情况应暂停扩展：

- 30 页索引后 90 天曝光仍很低。
- 大量访问只看首屏，不操作工具。
- 算法可信度问题频繁。
- 用户集中要求工业功能，而免费工具站无法承担。
- 竞品 SERP 压制严重且长尾也无法进入。
- 维护成本超过内容/广告/affiliate 回报。

## 23. 竞争差异化

### 23.1 不与专业软件硬碰

不要试图在第一版打败：

- CutList Optimizer。
- OptiCutter。
- CutList Evolution。
- MaxCut。
- PolyBoard / OptiCut。
- CNC nesting 软件。

### 23.2 差异化入口

可以主打：

- Free core use。
- No signup。
- Runs in browser。
- Kerf clearly included。
- Printable PDF。
- Plywood + linear tools in one lightweight site。
- Long-tail project presets。
- Mobile-friendly workshop mode。
- Clear explanations for beginners。

### 23.3 真实机会

竞品功能强，但常见问题是：

- 免费限制。
- UI 偏旧。
- 初学者不懂字段。
- 输入格式要求严格。
- 解释不足。
- 页面 SEO 与具体项目长尾没有完全覆盖。
- 移动端阅读切割图不够友好。

本项目应利用这些缝隙，而不是做重型 SaaS。

## 24. 风险与应对

| 风险 | 严重度 | 应对 |
|---|---:|---|
| 算法排版错误导致用户切错材料 | 极高 | 保守算法、警告、测试、免责声明 |

| 移动端不好用 | 中高 | 结果图缩放、shop mode |
